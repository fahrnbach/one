## Overview

Why stop at reusable components when you can build a **universal developer toolbox**?

This feature combines **snippets**, **templates**, and **type dictionaries** into a modular viewer. Think of it as your personal pattern library — ready to power everything from tiny fixes to full-blown CMS scaffolding.

---

## ✨ Key Features

- 🔍 **Fuzzy searchable** (powered by the same Angular viewer system)
- 🧱 **Snippets**: logic fragments, functions, hooks, helpers, regexes
- 📄 **Templates**: blog schemas, type maps, layout skeletons
- 🧬 **Type Dictionaries**: shared enums, object maps, TypeScript utility types
- 🔗 **Relation linking** to components or features using these tools

<Aside type="tip">Snippets are for logic. Templates are for structure. Types define the glue between them.</Aside>

---

## 🧰 Example Use Cases

- `fadeInOutAnimation`: Angular animation utility
- `jwtHelper`: Token parser with expiration check
- `postSchemaZod`: Validation template for CMS posts
- `postSchemaSQL`: Field-level template for SQLite schema
- `typeMap.ts`: Central reference for reusable enums or field type rules

---

## 🧠 Philosophy

Rather than scattering logic and templates across your project or docs, this structure:

- Centralizes developer wisdom
- Encourages **reuse-by-default**
- Enables intelligent UI linking and onboarding

<Aside type="caution">Always document your snippets! A line of context now saves hours of head-scratching later.</Aside>
